헤더(열 제목)가 무엇이든 관계없다.
헤더는 참고 힌트일 뿐이며, 컬럼의 역할은 반드시 실제 셀 내용을 보고 판단한다.

헤더가 "내용", "KR", "-", "텍스트", 공백, 혹은 아무 의미 없는 값이어도:
→ 셀에 긴 한국어 문장이 들어있으면 col_dialogue_rec
→ 셀에 짧은 한국어 감정/어투 묘사가 들어있으면 col_emotion_rec
→ 셀에 짧은 한국어 인명이 들어있으면 col_char_rec

헤더 키워드 매칭에 실패해서 중요한 열을 -1로 처리하는 것은 오류다.
반드시 각 열의 실제 데이터를 분석해서 역할을 결정하라.

1. 시트명이 정확히 '설정' → Type_명방캐릭터
2. 타임코드 컬럼 (HH:MM:SS.mmm --> HH:MM:SS.mmm 형식 값) → Type_무한대
3. 流程节点 헤더 또는 해당 컬럼 존재 → Type_전투
4. 台詞番号/ボイス名 헤더 OR (헤더에 ID + 功能 + 终选语音 조합) → Type_캐릭터음성
5. [병합셀 패턴] 캐릭터명 열에서 헤더 바로 다음 첫 데이터 행에만 값이 있고

6. 헤더에 时长/≤XXs 패턴 OR 파일명 열 없고 화자+대사 구조 → Type_PV
7. 파일명 열이 있고 캐릭터명 열이 매 행마다 채워진 여러 캐릭터 대화 구조 → Type_메인
8. 위 어디도 해당없음 → SKIP

■ col_dialogue_rec (녹음 대사 열) ← 가장 중요

판단 기준 (헤더 무관, 셀 내용으로만 판단):
① 해당 열의 셀들이 녹음 언어로 된 긴 문장을 포함한다
   · KR: 10글자 이상의 한국어 문장 (가-힣)
   · JP: 히라가나/가타카나/한자가 섞인 문장
   · EN: 5단어 이상의 영어 문장
② 여러 행에 걸쳐 지속적으로 대사가 채워진 열
③ "(호흡)", "[호흡]", "eff(...)", "(기합)" 같은 지문 표기가 섞여있으면 대사 열 확정

헤더 예시 — 모두 col_dialogue_rec일 수 있다:
"KR", "내용", "대사", "한국어", "-", "(공백)", "Contents", "Text", "Script", "Line", "台词KR", "韩文"
→ 헤더가 무엇이든 위 셀 내용 조건을 충족하면 col_dialogue_rec

col_dialogue_cn: 같은 원칙으로, 중국어 긴 문장이 있는 열

■ col_char_rec / col_char_cn (캐릭터명 열)

판단 기준 (셀 내용 우선):
① 셀 값이 짧다 (1~6글자 정도의 인명)
② 여러 행에 걸쳐 동일한 인명들이 반복 등장
③ 대사 열과 같은 행에서 화자를 식별할 수 있는 위치

· col_char_rec: 녹음 언어 인명 (KR이면 한국어 이름)
· col_char_cn: 중국어 인명 (配音对象/角色 또는 한자 이름)
· "???": 같은 시트에 실제 녹음 대상 캐릭터명 열이 별도로 있으면 그 열을 col_char_rec으로 사용
· "<player>", "玩家", "유저": 플레이어 캐릭터, 그대로 사용

■ col_emotion_rec / col_emotion_cn (감정/어투 열)

판단 기준 (셀 내용 우선):
① 셀 값이 짧은 행동·감정 묘사 문구 (3~30글자)
② 대사처럼 길지 않고, 이름처럼 짧지도 않은 중간 길이
③ 예: "뛰면서", "조금 엄숙하게", "愤怒地说", "落ち込んで", "미소 지으며 인사"
④ 헤더가 "말투", "감정", "语气备注", "詳細", "配音备注", "-", "(공백)" 이어도 해당

· col_emotion_cn: 한자(중국어) 감정 묘사가 주를 이루는 열
· col_emotion_rec: 녹음 언어(KR/JP/EN) 감정 묘사가 있는 열
· CN 감정 열과 rec 감정 열이 모두 있으면 반드시 구분할 것
  — 중국어 열을 col_emotion_rec로 지정하는 것은 오류

■ col_filename (파일명 열)

판단 기준 (셀 내용 우선):
① 행마다 고유한 값 (같은 값이 연속 반복되는 열은 제외)
② 파일명 패턴: Ex_vo_, Play_VO_, plotvoc_, 语音命名 접두사, 숫자+언더스코어 조합
③ 헤더: 语音命名/説明や注意事項/StringKey/录制ID 등

주의:
· 여러 행이 동일한 값 공유 → 씬/시퀀스 ID이므로 col_filename 금지
· "CBT2" 포함 열 → 구버전, col_filename 금지

■ col_optical (옵티컬 열)

· optical=EN이면 영어 긴 문장이 있는 열 (col_dialogue_rec와 다른 열)
· optical=CN이면 중국어 긴 문장이 있는 열
· 절대 녹음 대상 언어 대사 열을 여기에 넣지 말 것

■ col_duration (허용 시간 열)

판단 기준 (셀 값 패턴):
· "≤3s", "2.5s", "3초", "≤5.0s" 형태의 값이 여러 행에 등장 → col_duration
· 헤더가 무엇이든 위 패턴 값이 있으면 duration 열

■ col_step_type
· 步骤/画面/特效/头像对话 값이 있는 열 (행 필터용)
· skip_step_values: 画面/特效 포함. 旁白/내레이션은 절대 포함 금지

■ col_alt: alt/Alt/ALT/예비 — 헤더 또는 셀 내용 패턴으로 판단
■ col_functional: 功能/ボイス名 열
■ col_adr_wild: 是否有时间限制/Time Limit 열
■ col_timecode: 시간코드 형식(HH:MM:SS.mmm)이 있는 열

■ char_name_kr
· Type_캐릭터음성: 파일명/시트명에서 한국어 캐릭터명 추출
