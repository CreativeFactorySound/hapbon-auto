반드시 순수 JSON만 출력하세요. 마크다운 코드블럭(```), 설명 텍스트 절대 불가.

[타입 분류 우선순위 — 앞 조건 해당 시 뒤는 즉시 결론]
1. 시트명이 정확히 '설정' → Type_명방캐릭터
2. 타임코드 컬럼 (HH:MM:SS.mmm --> HH:MM:SS.mmm 형식 값) → Type_무한대
3. 流程节点 헤더 또는 해당 컬럼 존재 → Type_전투
4. 台詞番号/ボイス名 헤더 OR (헤더에 ID + 功能 + 终选语音 조합) → Type_캐릭터음성
5. [병합셀 패턴] 캐릭터명/キャラ/配音对象 열에서 헤더 바로 다음 첫 데이터 행에만 값이 있고, 그 뒤 여러 연속 행에서 해당 열이 비어 있다면 → 원본 엑셀에서 셀이 병합(merged cell)된 구조 → Type_짧은음성
   예: row5:[1]カリーナ  row6:(キャラ열 없음)  row7:(キャラ열 없음)  ... → Type_짧은음성
   - 파일명 열(語音命名/説明や注意事項 등) 이 존재해도 해당
   - 캐릭터가 2명 이상이어도, 대사가 짧지 않아도 해당
   - Type_메인과의 핵심 차이: Type_메인은 캐릭터명 열이 거의 모든 대사 행에 채워짐
6. 헤더에 时长/预期秒数/≤XXs 패턴 OR 파일명 열이 없고 화자+대사 구조 → Type_PV
7. 파일명 열이 명확히 있고 (语音命名/Ex_vo_/Play_VO_ 등), 캐릭터명 열이 매 대사 행마다 채워진 여러 캐릭터 대화 구조 → Type_메인
8. 위 어디도 해당없음 → SKIP

[Type_짧은음성 vs Type_메인 — 가장 중요한 구별]
미리보기 데이터에서 캐릭터명/キャラ/配音对象 열 패턴을 확인한다.
- 헤더 이후 첫 데이터 행에 캐릭터명이 있고, 뒤이은 여러 행에서 그 열이 비어 있다 → 반드시 Type_짧은음성.
- Type_메인은 각 대사 행마다 캐릭터명 열이 채워져 있어야 한다 (예: row5:[1]캐릭터A  row6:[1]캐릭터B ...).
- 파일명 열(説明や注意事項/語音命名 등)이 있어도 캐릭터명이 병합셀 패턴이면 Type_짧은음성.

[핵심 구별 포인트]
- Type_메인 vs Type_캐릭터음성: Type_메인은 여러 캐릭터(배우)가 등장하는 스토리 대본. Type_캐릭터음성은 파일/시트가 캐릭터 1명 전용이고 ID+功能+终选语音 구조.
- Type_메인: 헤더에 配音对象(캐릭터명)과 对话文本(대사) 쌍이 있고, 파일명(语音命名) 열 존재. 行마다 다른 캐릭터 등장.
- 旁白(내레이션) 행: 스킵이 아닌 포함 대상 — skip_step_values에 旁白 넣지 말 것.
- 画面, 特效 행: 씬 설명이지 대사가 아니므로 skip_step_values에 포함.

[즉시 SKIP 대상]
- 녹음 받는 언어(record)에 해당하는 대사가 전혀 없음 (KR이면 한국어, JP면 일본어, EN이면 영어)
- 美术需求, 音效需求, ID索引用 등 비대본 관리 시트
- 녹음 언어와 다른 언어 전용 시트 (예: KR 녹음인데 日文/家园线 日文 시트)

[헤더 행 탐색]
row0~row7 중 아래 키워드가 가장 많이 포함된 행. 헤더 위 행들은 프로젝트 개요/PART 설명이므로 무시.
키워드: 配音对象/对话文本/语音命名/流程节点/ID/功能/终选语音/台詞番号/KR Text/角色/台词/韩文/화자/캐릭터/キャラ/台詞/时间戳/TimeCode

[컬럼 인덱스 추론 — column_inference.md 기준, 0-based]

■ col_filename (파일명 열)
- 语音命名/Ex_vo_/Play_VO_/plotvoc_/説明や注意事項 헤더 OR 해당 패턴으로 시작하는 값
- 행마다 고유한 값 → 파일명 열 가능성 높음
- 여러 행이 동일한 값을 공유하는 열 → 씬/시퀀스 그룹 ID이므로 col_filename 금지 (-1로 처리)
- "CBT2" 포함 열 → 구버전 파일명 열, col_filename으로 지정 금지 (-1로 처리)
- 파일명 열이 여러 개인 경우: 행마다 고유한 값 중 납품 규격에 가장 가까운 것을 선택, 나머지는 매핑 제외

■ col_char_rec / col_char_cn (캐릭터명 열)
- col_char_rec: 배음대상/더빙 대상/캐릭터/화자/キャラ/Character 헤더 OR 녹음 언어(KR=한국어, JP=일본어, EN=영어) 인명이 있는 열
- col_char_cn: 配音对象/角色 헤더 OR 중국어 인명이 있는 열
- "???" 표기가 있는 열: 게임 내 숨김 처리된 캐릭터. 단, 같은 시트에 실제 녹음 대상 캐릭터명(한국어/영어)이 있는 별도 열이 있다면, 그 열을 col_char_rec으로 우선 사용할 것. "???" 열만 있다면 해당 열을 col_char_rec으로 사용.
- "<player>", "玩家", "유저" → 플레이어 캐릭터, 그대로 사용
- 동일 값이 여러 행 연속 (병합셀 흔적) → Type_짧은음성 신호

■ col_dialogue_rec / col_dialogue_cn (대사 열)
- col_dialogue_rec: 녹음 받는 언어의 주 대사 열. KR이면 가장 긴 한국어 문장 열, JP이면 台詞 등 일본어 대사 열, EN이면 영어 대사 열. 절대 col_dialogue_cn이나 col_optical에 넣지 말 것.
  · 헤더 키워드: 対話文本/终选语音/台詞/台词/대사/내용/Contents/Dialogue/Text/Script/Line
  · 헤더가 "내용"인 열 → 한국어 대사가 포함되어 있으면 col_dialogue_rec로 판단할 것
- col_dialogue_cn: 对话文本/终选语音/中(...) 등 중국어 원문 대사 열. JP 스크립트에 중국어가 없으면 -1.
- "eff(...)", "(호흡 지문)", "(기합)" 형태 → 대사 열에 포함되는 효과음/호흡 표기이므로 대사 열 판단에 포함

■ col_emotion_rec / col_emotion_cn (감정 열)
- 헤더 키워드: 말투/감정/詳細/语气/配音备注/情绪/状態/语音备注 등
- 헤더가 달라도(화면/画面/步骤 등) 셀 값들이 짧은 감정·행동 묘사 문장이면 감정 열로 판단
  예: "露米叹气", "愤怒地说", "打滚耍赖的样子", "落ち込んで", "뛰면서", "조금 엄숙하게"
  → 연출 씬 설명("PART1 전투 진입", "화면 전환")과 구별할 것

- [핵심 규칙] CN 감정 열과 rec(녹음 언어) 감정 열이 모두 있는 경우 반드시 구분할 것:
  · col_emotion_cn: 한자(중국어) 값이 주를 이루는 열 (예: 语气备注 원본)
  · col_emotion_rec: 녹음 언어(KR=한국어, JP=일본어, EN=영어) 값이 있는 열
  · 헤더에 "KR" / "韩" / "한국어" / "JP" / "日" / "EN" 접미사가 있으면 무조건 col_emotion_rec
  · 두 열 모두 존재할 때 중국어 열을 col_emotion_rec로 지정하는 것은 오류
  · 셀 값의 언어를 직접 확인: 녹음 언어 텍스트 포함 열 → col_emotion_rec, 한자 위주 열 → col_emotion_cn

■ col_optical (옵티컬 열)
- optical=EN이면 영어 대사 열, optical=CN이면 중국어 대사 열. 해당 언어 열 없으면 -1.
- 절대 녹음 대상 언어 대사를 여기에 넣지 말 것.
- EN 열이 두 개인 경우: 앞의 EN 열을 col_optical로, 뒤 중복 열은 매핑 제외

■ col_duration (duration 열)
- 时长/预期秒数 헤더 OR ≤XXs 형식 값
- 소수점 숫자(3.0, 2.5 등) → 초 단위 duration일 가능성 있음. 인접 열 맥락으로 판단

■ col_step_type (스텝 타입 열)
- 步骤/画面/特效/头像对话 값이 있는 열 (행 필터용)
- skip_step_values: 画面/特效/头像对话 등 대사가 아닌 연출 타입값 → 여기에 포함
- 旁白/ナレーション/Narration/내레이션 → skip_step_values에 절대 넣지 말 것 (포함 대상)

■ col_alt: Alt/alt/ALT/예비 헤더 열
■ col_functional: 功能/ボイス名 열 (Type_캐릭터음성에서 감정 앞 참고 정보로 포함)
■ col_adr_wild: 是否有时间限制/Time Limit 열 (Type_무한대)
■ col_timecode: 时间戳/TimeCode 열 (Type_무한대)

■ char_name_kr
- Type_캐릭터음성: 파일명/시트명에서 한국어 캐릭터명 추출
- Type_PV: 시트 전체가 특정 1명 전용이면 해당 캐릭터명 추출

[표준 매핑에 포함하지 않을 열 — extra_cols로 자동 보존됨]
